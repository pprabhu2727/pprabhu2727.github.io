3D Video Game

This video game was completed in a team of two over five months. The game is a zombie survival game where the player's goal is to survive for as long as possible while eliminating all enemies. This game was built off of a bare-bones game engine known as TAGE, so most of the features in the game were built from scratch using Java.

Features Implemented

  • Input Handling
    • Based on specific inputs, different actions would take place within the game. I created an input handler to link an input key from a controller, keyboard, or mouse with an in-game action.
  • Models and Animation
    • All game models, animations, and textures were handmade and created in Blender with the help of skeletal rigging and texture mapping tools.
  • Networked Multiplayer
    • The game supports multiplayer for up to two players. The networking between the client and server was done using Java by having the client and server send UDP packets to each other whenever an action occurred in-game. The client and server also had a custom protocol to handle incoming packets and send them to a game controller to perform synchronized actions across all clients.
  • Scripting
    • Scripting was implemented to allow for a separate script file to initialize some game parameters. The program would read from the javascript file and initialize the correct parameters in the Java program to make the game-balancing process easier.
  • AI and NPC
    • Rangers and Monsters are non-playable characters (NPCs) that follow predetermined behavior patterns. The Ranger, for instance, is programmed with a behavior tree that instructs it to patrol the arena until a player approaches. Once a player is detected, the Ranger's behavior changes: it becomes aggressive, enlarges in size, emits an attack sound, and attacks the player. The Ranger follows a circular path around the arena to perform its patrolling behavior. This is achieved by converting its Cartesian coordinates to polar coordinates, calculating its path based on the polar coordinates, and then converting back to Cartesian coordinates. This process results in the Ranger rotating around the arena in a circular motion.
  • Physics
    • The ranger's attack implements physics and uses the JBullet physics engine. The ranger's grenade attacks will bounce off other grenades, the floor, and the avatar. When the game detects the collision between a grenade and the player, the player's health will be decreased.
  • Skybox, Terrain, Lighting, Camera Controller, Hierarchical Scenegraph, and 3D Sound
